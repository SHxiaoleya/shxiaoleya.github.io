我的第一个静态网站搭建历程

"种一棵树最好的时间是十年前，其次是现在。" 建立个人网站也是如此。从购买域名到最终部署上线，这个过程就像培育一棵树，需要耐心、规划和持续的努力。

大家好，我是树小乐。今天我想和大家分享我从零开始搭建个人网站 树.top 的完整过程。作为一个前端开发爱好者，我一直想要一个属于自己的数字空间，记录我的学习历程和技术思考。

为什么选择静态网站？

在开始搭建之前，我面临一个重要的选择：使用动态网站（如WordPress）还是静态网站？经过仔细考量，我选择了静态网站，主要原因如下：

  • 性能优势：静态网站加载速度快，不需要数据库查询和服务器端渲染
  • 安全性高：没有数据库，减少了被攻击的风险
  • 成本低廉：可以利用GitHub Pages等免费服务托管
  • 版本控制友好：所有内容都可以用Git管理，方便回溯和协作
  • 学习价值：可以更深入地理解前端技术和部署流程

图：代码编辑器界面 - 静态网站开发的核心工具

技术栈选择

对于静态网站，有多种技术方案可供选择。我最终选择了最基础的HTML/CSS/JS方案，原因如下：

  1. 简单直接：不需要构建工具，编写后直接可以预览
  2. 完全控制：每一行代码都由自己编写，便于定制和调试
  3. 学习基础：对于前端新手来说，这是最好的学习方式
  4. 兼容性好：纯HTML在所有浏览器上都能完美运行

基础HTML结构示例

下面是我网站首页的基础HTML结构：

CSS设计思路

在设计网站样式时，我选择了自然、简洁的风格，以配合"树"这一主题。主要的设计原则包括：

1. 色彩方案

以绿色为主色调，搭配暖金色作为点缀，营造自然、温暖的视觉效果。


/* 色彩定义 */
:root {
    --primary-green: #2d5a27;
    --light-green: #8db596;
    --accent-gold: #d4af37;
    --light-bg: #f9f7f2;
    --dark-text: #333333;
}

/* 响应式设计 */
@media (max-width: 768px) {
    .container {
        padding: 0 1.5rem;
    }
    
    h1 {
        font-size: 2.2rem;
    }
}

/* 动画效果 */
@keyframes fadeIn {
    from { opacity: 0; transform: translateY(20px); }
    to { opacity: 1; transform: translateY(0); }
}

.fade-in {
    animation: fadeIn 0.6s ease-out;
}

2. 响应式布局

确保网站在手机、平板和电脑上都有良好的显示效果。

JavaScript交互功能

为了让网站更具交互性，我添加了一些基本的JavaScript功能：


// 移动端菜单切换
document.getElementById('menuToggle').addEventListener('click', function() {
    document.getElementById('navLinks').classList.toggle('active');
});

// 回到顶部按钮
const backToTopButton = document.getElementById('backToTop');

window.addEventListener('scroll', function() {
    if (window.pageYOffset > 300) {
        backToTopButton.classList.add('visible');
    } else {
        backToTopButton.classList.remove('visible');
    }
});

backToTopButton.addEventListener('click', function() {
    window.scrollTo({
        top: 0,
        behavior: 'smooth'
    });
});

// 表单验证示例
document.getElementById('contactForm').addEventListener('submit', function(e) {
    const email = document.getElementById('email').value;
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    
    if (!emailRegex.test(email)) {
        e.preventDefault();
        alert('请输入有效的电子邮件地址');
        return false;
    }
});

部署到GitHub Pages

将网站部署到GitHub Pages的过程非常简便：

  1. 在GitHub上创建新的仓库
  2. 将本地代码推送到仓库
  3. 在仓库设置中启用GitHub Pages
  4. 选择部署分支（通常是main或gh-pages）
  5. 等待几分钟，网站就会上线

对于自定义域名（如树.top），还需要额外配置：


# 1. 在仓库根目录添加CNAME文件
echo "树.top" > CNAME

# 2. 将CNAME文件提交到仓库
git add CNAME
git commit -m "添加自定义域名配置"
git push

# 3. 在域名注册商处添加DNS记录
# 类型：CNAME
# 名称：@ 或 www
# 值：用户名.github.io

遇到的挑战与解决方案

1. 中文字体加载问题

最初我使用了本地字体文件，但文件体积较大，影响了加载速度。解决方案是使用Google Fonts的Noto Sans SC字体，它专门为中文优化，并且通过CDN分发，速度更快。

2. 移动端菜单体验

最初设计的菜单在移动端上体验不佳。通过添加汉堡菜单图标和优化触摸反馈，显著改善了移动端用户体验。

3. 图片优化

网站中的图片最初没有优化，导致加载缓慢。通过压缩图片、使用WebP格式和懒加载技术，显著提升了页面性能。

总结与展望

通过这次静态网站搭建项目，我不仅获得了一个展示个人品牌的平台，还巩固了前端开发的基础知识。整个过程让我深刻体会到：

  • 简单就是美：不需要复杂的技术栈，基础HTML/CSS/JS也能创建出色的网站
  • 迭代改进：网站不是一次完成的，需要不断优化和更新
  • 学习永无止境：每个问题都是学习的机会

未来，我计划为网站添加以下功能：

  1. 博客系统，支持文章分类和标签
  2. 评论功能，方便与读者互动
  3. 暗色模式，提供更舒适的夜间阅读体验
  4. 搜索功能，帮助读者快速找到内容
  5. RSS订阅，方便读者跟踪更新

感谢你阅读这篇文章。如果你也在考虑建立自己的个人网站，我希望我的经验能对你有所帮助。记住，最重要的是开始行动，然后持续改进。
